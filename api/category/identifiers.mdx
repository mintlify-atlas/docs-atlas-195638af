## Overview

Category samples in HealthKit represent data that fits into a predefined set of values. This page lists all 67 available category type identifiers.

## Read-Only Categories

These category types are automatically tracked by Apple Watch or iOS and cannot be written by third-party apps.

<AccordionGroup>
  <Accordion title="HKCategoryTypeIdentifierAppleStandHour" icon="apple">
    Tracks whether the user stood during a particular hour.
    
    **Values**: `CategoryValueAppleStandHour`
    - `stood` (0)
    - `idle` (1)
    
    **Read-only**: Yes
  </Accordion>
  
  <Accordion title="HKCategoryTypeIdentifierHighHeartRateEvent" icon="heart-pulse">
    Records events when the user's heart rate exceeds a specified threshold.
    
    **Values**: `CategoryValueNotApplicable.notApplicable`
    
    **Read-only**: Yes
  </Accordion>
  
  <Accordion title="HKCategoryTypeIdentifierLowHeartRateEvent" icon="heart">
    Records events when the user's heart rate falls below a specified threshold.
    
    **Values**: `CategoryValueNotApplicable.notApplicable`
    
    **Read-only**: Yes
  </Accordion>
  
  <Accordion title="HKCategoryTypeIdentifierHeadphoneAudioExposureEvent" icon="headphones">
    Tracks when headphone audio exposure exceeds safe listening levels.
    
    **Values**: `CategoryValueNotApplicable.notApplicable`
    
    **Read-only**: Yes
  </Accordion>
</AccordionGroup>

## Type Reference

### CategoryValueSeverity

```typescript
enum CategoryValueSeverity {
  notPresent = 1,
  mild = 2,
  moderate = 3,
  severe = 4,
  unspecified = 0,
}
```

### CategoryValuePresence

```typescript
enum CategoryValuePresence {
  notPresent = 1,
  present = 0,
}
```

### CategoryValueNotApplicable

```typescript
enum CategoryValueNotApplicable {
  notApplicable = 0,
}
```
