## Read-only vs Writeable

Most quantity types are **writeable**, meaning third-party apps can save data to them using `saveQuantitySample()`. However, some types are **read-only** and can only be written by Apple's own apps and devices:

- `HKQuantityTypeIdentifierAppleExerciseTime`
- `HKQuantityTypeIdentifierAppleStandTime`
- `HKQuantityTypeIdentifierWalkingHeartRateAverage`
- `HKQuantityTypeIdentifierAppleWalkingSteadiness`
- `HKQuantityTypeIdentifierAtrialFibrillationBurden`

The TypeScript types distinguish between these with `QuantityTypeIdentifierWriteable` and `QuantityTypeIdentifierReadOnly`.

## iOS Version Availability

Some quantity types are only available on newer iOS versions:

- **iOS 11+**: Heart rate variability, insulin delivery, walking heart rate average
- **iOS 13+**: Environmental and headphone audio exposure
- **iOS 14+**: Mobility metrics, six-minute walk test
- **iOS 14.5+**: Apple Move Time
- **iOS 15+**: Walking steadiness, number of alcoholic beverages
- **iOS 16+**: Running metrics, heart rate recovery, underwater depth, water temperature, environmental sound reduction
- **iOS 17+**: Cycling metrics, time in daylight, physical effort, sleeping wrist temperature
- **iOS 18+**: Additional distance sports, workout effort scores, sleeping breathing disturbances

<Tip>
  Use `isObjectTypeAvailable()` to check if a specific quantity type is available on the current device before requesting authorization or querying data.
</Tip>
